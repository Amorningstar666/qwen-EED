## 2. Descripción Narrativa (Formato PGN)

### **Como** Juez Disciplinario o Revisor autorizado,  
### **Quiero** revisar, modificar (si es necesario) y aprobar o rechazar el proyecto de Auto de Suspensión Provisional remitido por el Instructor,  
### **Para** garantizar que la medida cautelar cumpla estrictamente con los requisitos legales (motivos taxativos, proporcionalidad y consulta obligatoria) antes de su notificación y ejecución, asegurando el debido proceso.

---

## 3. Contexto y Reglas de Negocio

### 3.1. Flujo de Trabajo
1. El Juez recibe en su bandeja de "Pendientes de Revisión" los autos proyectados por los Instructores.
2. El sistema valida que el Instructor haya marcado la casilla **"Enviada a Consulta"** (requisito habilitante desde HU-17-01).
3. El Juez visualiza el detalle: investigado, cargos, motivos art. 46, duración propuesta y fecha de consulta al Superior.
4. El Juez puede:
    *   **Aprobar:** Firma digitalmente y envía a notificación.
    *   **Modificar:** Ajusta motivación, duración o fundamentos antes de aprobar.
    *   **Rechazar:** Devuelve al instructor con observaciones (la suspensión no se decreta).

### 3.2. Reglas de Negocio Críticas
*   **RN-01 (Consulta Previa):** No se puede aprobar ninguna suspensión provisional si el sistema no registra la fecha/hora de envío a consulta al Superior Jerárquico (Art. 47 CGD).
*   **RN-02 (Motivación):** El Juez debe verificar que los hechos descritos se subsuman claramente en uno de los 5 causales del Art. 46 CGD.
*   **RN-03 (Duración):** La duración total (inicial + prórroga) no puede exceder los límites legales sin una justificación excepcional documentada.
*   **RN-04 (Segregación de Funciones):** El Juez que revisa no puede ser el mismo usuario que proyectó el auto (validación RBAC cruzada).
*   **RN-05 (Trazabilidad):** Cualquier modificación realizada por el Juez sobre el texto original del Instructor debe quedar registrada en el historial de versiones del documento.

## 4. Criterios de Aceptación (Gherkin)

### Escenario 1: Revisión Exitosa y Aprobación (Flujo Feliz)
| Dado | Que existe un Auto de Suspensión proyectado por el Instructor con estado "En Revisión" |
| :--- | :--- |
| **Y** | El auto cuenta con la marca de verificación "Enviada a Consulta" y fecha de radicación ante el Superior |
| **Y** | Los motivos seleccionados corresponden taxativamente al Art. 46 CGD |
| **Cuando** | El Juez revisa el contenido, verifica la legalidad y presiona el botón "Aprobar y Firmar" |
| **Entonces** | El sistema cambia el estado a "Aprobado - Listo para Notificación" |
| **Y** | Se genera el PDF final con la firma digital del Juez |
| **Y** | Se registra en auditoría: "Auto aprobado por [Usuario Juez] el [Fecha]" |

### Escenario 2: Bloqueo por Falta de Consulta Obligatoria
| Dado | Un Instructor proyectó un auto pero olvidó marcar/validar la "Enviada a Consulta" |
| :--- | :--- |
| **Cuando** | El Juez intenta aprobar el auto |
| **Entonces** | El sistema muestra una alerta crítica: "Error: No se ha registrado la Consulta al Superior Jerárquico (Art. 47 CGD)" |
| **Y** | El botón "Aprobar" permanece deshabilitado |
| **Y** | El sistema sugiere: "Devolver al Instructor para subsanar" |

### Escenario 3: Modificación del Proyecto por Parte del Juez
| Dado | El Juez considera que la redacción de los motivos es insuficiente o la duración es excesiva |
| :--- | :--- |
| **Cuando** | El Juez edita los campos de "Motivación" o "Duración" en modo edición |
| **Y** | Ingresa una justificación para el cambio (ej: "Ajuste de duración según principio de proporcionalidad") |
| **Y** | Presiona "Guardar Cambios y Aprobar" |
| **Entonces** | El sistema guarda una versión modificada del documento |
| **Y** | El PDF final refleja los cambios del Juez, marcando las diferencias en el historial |
| **Y** | Se notifica al Instructor sobre las modificaciones realizadas |

### Escenario 4: Rechazo del Auto de Suspensión
| Dado | El Juez determina que no se configuran los presupuestos para la suspensión |
| :--- | :--- |
| **Cuando** | El Juez selecciona la opción "Rechazar Proyecto" e ingresa las observaciones |
| **Entonces** | El estado del trámite regresa a "Bandeja del Instructor" con marca "Rechazado" |
| **Y** | No se genera ningún acto administrativo de suspensión |
| **Y** | El Investigado NO es suspendido en este acto |

### Escenario 5: Validación de Segregación de Funciones (RBAC)
| Dado | Un usuario intenta revisar un auto que él mismo proyectó como Instructor |
| :--- | :--- |
| **Cuando** | El usuario accede a la tarea de revisión |
| **Entonces** | El sistema oculta la opción de "Aprobar" o muestra mensaje: "No puede revisar sus propios actos (Principio de Separación de Funciones)" |

## 5. Requisitos de Interfaz y Experiencia de Usuario (UI/UX)

### 5.1. Visualización de Datos
*   **Panel de Lectura:** Vista clara del proyecto original (solo lectura) vs. Panel de Edición (habilitado solo si el Juez decide modificar).
*   **Alertas Visuales:**
    *   🟢 **Check Verde:** "Consulta a Superior Verificada" (Con fecha y hora).
    *   🔴 **Alerta Roja:** Si falta algún requisito legal (motivos, fechas).
*   **Comparador de Versiones:** Si el Juez edita, mostrar en pantalla dividida: "Texto Original (Instructor)" vs "Texto Propuesto (Juez)".

### 5.2. Controles de Acción
*   **Botón "Aprobar y Firmar":** Habilitado solo si `Check_Consulta == TRUE` Y `Motivos_Validados == TRUE`.
*   **Botón "Modificar":** Abre editor de texto enriquecido con control de cambios.
*   **Botón "Rechazar":** Abre modal obligatorio para ingresar causal de rechazo y comentarios.

### 5.3. Accesibilidad
*   Navegación completa por teclado para revisión de documentos largos.
*   Contraste alto para alertas de bloqueo legal.

## 6. Trazabilidad Normativa (Código General Disciplinario - Ley 1952/2019 y Modificatorias)

| Artículo | Norma | Aplicación en la HU |
| :--- | :--- | :--- |
| **Art. 46** | Causales de Suspensión | El Juez debe validar estrictamente que los hechos encajen en estas causales (Peligro de fuga, obstrucción, etc.). |
| **Art. 47** | Consulta Obligatoria | **Punto Crítico:** El Juez verifica que se haya consultado al Superior Jerárquico antes de firmar. Sin esto, el acto es nulo. |
| **Art. 48** | Efectos de la Suspensión | El Juez define claramente en el auto qué cargos específicos motivan la suspensión y los efectos inmediatos. |
| **Art. 12** | Ley 2094/2021 | Garantiza que quien investiga (Instructor) no sea quien decide la medida cautelar definitiva (Juez), asegurando imparcialidad. |
| **Art. 9** | Debido Proceso | La revisión judicial actúa como garantía de defensa antes de la ejecución de la medida. |
